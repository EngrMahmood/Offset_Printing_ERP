Release Notes ({{ erp_software_version }})

  • Removed department from SKU master; `PlanningJob.department` is now sourced from PO intake payload.
  • Approving a master SKU (from any UI path) now auto-syncs matching New PO lines into Planning Jobs.
  • Added `size_w_mm` and `size_h_mm` fields to Pending SKU master entry to match the master edit form.
  • Quick-config modal feature has been removed in this release.

This guide explains the planning workflow in simple language. Follow the sequence exactly so repeat and new SKU jobs are routed correctly.

1. Master SKU (Step 1)

  • Create or bulk upload SKU master records.
  • Use the workflow: Draft → Reviewed → Approved.
  • Only approved master records should be used downstream in planning.
  • If a SKU is missing required master fields, it will remain in Pending SKU until complete.

2. PO Intake (Step 2)

  • Open the PO Intake Queue from the Planning home page.
  • You can either upload a PO PDF or use Manual PO Entry for non-PDF orders.
  • After upload, the system redirects you to the PO Review page for verification before intake.
  • On PO Review you can correct the PO number, add missing PO lines manually, or ignore unwanted SKUs.
  • Manual PO entry supports the same fields as PDF extraction: PO number, PO date, approval date, department, delivery location, supplier, buyer, grand total, SKU, job name, quantity, unit, delivery date, unit cost, and net total.
  • Repeat lines are routed directly to Planning Jobs after sync.
  • New lines go to Pending SKU for master completion and approval.
  • Ignored SKUs are tracked on the PO and can be restored later from the Ignored SKUs page.

3. Pending New SKU (Step 3)

  • Complete missing master fields for new SKUs that arrive from PO Intake.
  • Use the Pending SKU workflow to edit and approve new SKU master data.
  • After approval, new SKU lines auto-sync into Planning Jobs.
  • Job Name is normally taken from the PO and should not be changed unless necessary.
  • Application values allowed: UV, Lamination Gloss, Lamination Matt, NO.
  • Purchase Material Origin values allowed: Local, Imported.

4. Planning Jobs (Step 4)

  • Review Planning Jobs, filter by status, department, machine, and PO.
  • Use the bulk action panel to update statuses or archive selected jobs.
  • You can place jobs on hold, release hold, archive, or permanently delete jobs.
  • Archived jobs are moved to the Archived Jobs page for restore or permanent delete.
  • Always verify the job details before approving or archiving to avoid data loss.

5. Approval Queue (Step 5)

  • Jobs flow through: Draft → Pending Review → Pending Approval (Manager).
  • Only approved jobs should be released to execution and production planning.
  • Use this queue to ensure master data and routing are correct before final release.

6. Job Card and Scan (Step 6)

  • Print the A4 Job Card for shop floor operations.
  • Use the QR scan feature to open and track jobs quickly on the production floor.
  • Job Card version updates after each save on the planning job.

7. Cost / Department Mismatch Alerts

  • Repeat jobs highlight cost mismatches with a COST ALERT note.
  • Repeat jobs highlight department mismatches with a DEPARTMENT ALERT note.
  • Review these alerts before creating or updating Planning Jobs.

8. Professional Change Tracking

  • This release is documented in the project `CHANGELOG.md` file for audit and review.
  • Use the changelog to compare versions and verify the merged planning/main updates.
Keep this planning guide updated whenever workflow rules, mandatory fields, or routing behavior changes. Review the guide after each release or merge to ensure users have current instructions.
