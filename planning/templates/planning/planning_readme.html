Planning Module - Full Easy Guide

Generated on: {{ generated_on|date:'Y-m-d H:i' }}

This guide explains the planning workflow in simple language. Follow the sequence exactly so repeat and new SKU jobs are routed correctly.

1. Master SKU (Step 1)

  • Create or bulk upload SKU master records.
  • Use workflow: Draft -> Reviewed -> Approved.
  • Only approved master records should be used downstream.

2. PO Intake (Step 2)

  • Upload PO PDF and review extracted lines in intake queue.
  • Repeat lines go directly to Planning Jobs.
  • New lines go to Pending SKU master completion first.
  • Ignore any new SKU during PO review if it should not be sent to pending processing.
  • Ignored SKUs are stored with the PO payload and removed from the active pending/new counts.
  • Use the Ignored SKUs page to restore items back into pending processing when needed.

3. Pending New SKU (Step 3)

  • Complete missing master fields for new SKUs.
  • Job name is PO-derived; do not enter manually.
  • Application values allowed: UV, Lamination Gloss, Lamination Matt, NO.
  • Purchase Material Origin allowed: Local, Imported.
  • After approval, new SKU lines auto-sync to Planning Jobs.

4. Planning Jobs (Step 4)

  • Review, filter, edit, and manage planning jobs.
  • Use bulk status update only when appropriate.
  • Prepare jobs for QC and Production Manager approvals.

5. Approval Queue (Step 5)

  • Draft -> Pending Review -> Pending Approval (Manager).
  • Only approved jobs should be released to execution.

6. Job Card and Scan (Step 6)

  • Print A4 Job Card for shop floor.
  • Use QR scan to open and track jobs quickly.

7. Cost / Department Mismatch Alerts

  • Repeat jobs highlight cost mismatch against PO as COST ALERT.
  • Repeat jobs highlight department mismatch against PO as DEPARTMENT ALERT.
  • These alerts help planners review variances before production.
Keep this planning guide updated whenever workflow rules, mandatory fields, or routing behavior changes.
